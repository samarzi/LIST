# ТЗ — Система взаимной поддержки LIST
**Версия 1.0 | Telegram Mini App + Bot**

---

## 1. Обзор продукта

LIST — это Telegram-сервис взаимной поддержки в достижении целей. Пользователи помогают друг другу добиваться результатов: ставят публичные цели, доказывают результаты, получают поддержку от партнёра и оценку от сообщества.

**Ключевая идея:** каждый участник одновременно является учеником (ставит цели) и смотрящим (следит за чужими целями). Дополнительно — может стать учителем в своей теме.

**Внутренняя валюта:** LIT — используется как мера вклада и репутации, в будущем конвертируется в реальные деньги.

## 2. Технический стек

### Backend
- **Runtime:** Node.js 20+ или Python 3.11+ (FastAPI)
- **База данных:** PostgreSQL 15+ (основная) + Redis (кэш, очереди, сессии)
- **ORM:** Prisma (Node) или SQLAlchemy (Python)
- **Очереди задач:** BullMQ (Node) или Celery (Python)
- **Хостинг:** Railway или Render (старт), масштабирование на VPS

### Telegram
- **Bot:** Telegraf.js (Node) или python-telegram-bot / aiogram (Python)
- **Mini App (WebApp):** React + Vite или чистый HTML/CSS/JS
- **Telegram API:** Bot API 7.0+, Web Apps API

### Фронтенд (Mini App)
- **Фреймворк:** React или Vue 3
- **Стили:** Tailwind CSS — адаптив под мобайл и ПК обязателен
- **Telegram Web App SDK:** подключается через `telegram-web-app.js`
- **Состояние:** Zustand или Pinia

### Инфраструктура
- **Вебхуки:** HTTPS обязателен (Telegram требует SSL)
- **Хранилище файлов:** Cloudflare R2 или S3-совместимое (для доказательств)
- **Мониторинг:** Sentry (ошибки), LogSnag или собственные логи

## 4. Пользователи и роли

### 4.1 Регистрация

При первом запуске бота `/start`:
1. Бот запрашивает имя (или берёт из Telegram профиля)
2. Показывает краткое объяснение системы
3. Создаёт профиль в БД
4. Открывает Mini App для заполнения профиля

**Данные профиля:**
- `telegram_id` (уникальный)
- `username` (из Telegram)
- `display_name`
- `created_at`
- `level` (1–10, default: 1)
- `rating` (float, default: 5.0)
- `lit_balance` (int, default: 0)
- `total_goals_completed` (int)
- `total_goals_failed` (int)
- `is_teacher` (boolean, default: false)
- `teacher_rating` (float, nullable)
- `is_arbitrator` (boolean, default: false)
- `freeze_count_30d` (int) — количество заморозок за последние 30 дней

### 4.2 Роли

Каждый пользователь одновременно:

| Роль | Описание | Обязательность |
|---|---|---|
| **Ученик** | Ставит цели, делает чекины, доказывает результат | Обязательно |
| **Смотрящий** | Следит за прогрессом своего ученика, подтверждает | Обязательно |
| **Учитель** | Обучает в своей теме, принимает LIT за занятия | По желанию |
| **Арбитр** | Рассматривает жалобы, оценивает спорные цели | Назначается системой |

### 4.3 Система уровней (1–10)

Уровень растёт за:
- Выполненные цели (вес зависит от сложности)
- Успешные подтверждения как смотрящий
- Честное голосование (попал в медиану)
- Время в системе (небольшой пассивный рост)

```
Уровень 1–3   → новичок, учитель недоступен
Уровень 4–6   → средний, может стать учителем
Уровень 7–9   → опытный, доступ ко всем функциям
Уровень 10    → мастер, особый статус в профиле
```

**Арбитр** — автоматически назначается при:
- Уровень ≥ 6
- Рейтинг ≥ 4.0
- Выполнено ≥ 10 целей

Статус арбитра теряется при:
- Рейтинг упал ниже 3.5
- 3+ некорректных решения подряд (по апелляциям)

### 4.4 Рейтинг пользователя

Рейтинг (0–5, float) влияет на:
- Видимость в рекомендациях
- Доступность статуса арбитра и учителя
- Доверие сообщества

Факторы изменения рейтинга:

| Событие | Изменение |
|---|---|
| Цель выполнена (оценка > 5) | +0.1 |
| Цель провалена | -0.1 |
| Смотрящий подтвердил валидную цель | +0.05 |
| Смотрящий подтвердил фейк (жалоба подтверждена) | -0.3 |
| Жалоба пользователя подтверждена арбитром | -0.2 |
| Заморозка (2-я за месяц) | -0.05 |
| Заморозка (3-я+ за месяц) | -0.15 |
| Форс-мажор принят | 0 (штраф снимается) |

## 5. Система пар

### 5.1 Матчинг

При регистрации пользователь попадает в **очередь матчинга**.

**Алгоритм:**
1. Берём двух пользователей из очереди
2. Проверяем совместимость (уровень ±2, не были в паре раньше)
3. Создаём пару: пользователь А — смотрящий для Б, пользователь Б — смотрящий для А
4. Бот уведомляет обоих и отправляет ссылки на Telegram-аккаунты друг друга

**Важно:** пара асимметрична — у каждого свой смотрящий. Один человек одновременно является учеником (у него есть смотрящий) и смотрящим (он следит за кем-то другим).

**Структура пары в БД:**
```
pairs:
  id
  watcher_id    → user (смотрящий)
  student_id    → user (ученик)
  status        → active | frozen | ended
  created_at
  ended_at
```

### 5.2 Чекины

Ученик делает чекин боту каждый день:
- Отправляет сообщение / фото / видео боту
- Бот пересылает смотрящему или уведомляет его
- Смотрящий подтверждает (`✅ Принято` / `🔄 Уточни`)

**Если смотрящий не отвечает на чекин 24 часа** → автоматически подтверждается (защита ученика от неактивного смотрящего).

### 5.3 Смена пары

Инициирует **ученик**:
- Не чаще 1 раза в 2 недели
- Обязательно указывает причину
- Частые смены (3+ за 60 дней) отображаются в профиле как предупреждение для будущих партнёров

**После смены:**
- Старый смотрящий возвращается в очередь поиска ученика
- Ученик попадает в очередь поиска нового смотрящего

### 5.4 Механика заморозки при неактиве

**Триггер:** 3 дня без активности одного из участников пары (нет чекинов / ответов / голосований).

```
День 3 без активности
         ↓
Бот уведомляет активного партнёра:
"Твой [смотрящий/ученик] не выходил на связь 3 дня"

Активный партнёр выбирает:
  ⏳ Ждать → партнёр помечается как frozen
  🔄 Заменить → запускается автопоиск

Замороженный при входе видит:
  → партнёр ждал → предложение вернуться к паре
  → партнёр ушёл → пользователь в очереди
```

**Влияние на рейтинг заморозки:**

| Случай | Штраф |
|---|---|
| 1-я заморозка за 30 дней | нет |
| 2-я подряд | −0.05 к рейтингу |
| 3-я и далее | −0.15 к рейтингу |
| Форс-мажор с объяснением (принято арбитром) | штраф снимается |

## 6. Цели

### 6.1 Постановка цели

Ученик заполняет структурированную форму в Mini App:

```
Поле 1: Что конкретно сделаю?
  → короткий заголовок, ≤ 100 символов
  → пример: "Создам работающее веб-приложение с авторизацией"

Поле 2: Как измерю результат?
  → конкретные критерии, ≤ 300 символов
  → пример: "Приложение задеплоено на хостинге, есть ссылка, работает регистрация и вход"

Поле 3: Зачем мне это?
  → мотивация, ≤ 200 символов

Поле 4: План действий
  → шаги к цели, ≤ 500 символов

Поле 5: Дедлайн
  → дата

Поле 6: Тип доказательства
  → видео / скриншот / ссылка / документ / комбинация
```

**Валидация формы:** поля 1, 2, 5 обязательны. Если поле 2 содержит менее 30 символов или выглядит абстрактно — показываем предупреждение с примером.

**После отправки:** цель уходит смотрящему на оценку сложности.

### 6.2 Оценка сложности смотрящим

Смотрящий видит заполненную форму цели и выставляет оценку сложности:

**Ползунок 0–10** с якорными примерами:

| Балл | Пример цели |
|---|---|
| 0 | Невалидная цель (не описана конкретно) |
| 1–2 | Встать в 6 утра 7 дней подряд |
| 3–4 | Пробежать 5 км без остановки |
| 5–6 | Написать и опубликовать статью |
| 7–8 | Выучить 500 слов на иностранном языке |
| 9–10 | Создать и задеплоить веб-приложение |

**Оценка 0** означает: цель невалидна → возвращается ученику с комментарием.

После оценки:
- Ученик видит оценку и должен согласиться или оспорить
- При несогласии → арбитр рассматривает и выносит решение
- При согласии → цель становится активной (`в процессе`)

### 6.3 Статусы цели

```
черновик → на оценке → в процессе → на проверке → на голосовании → выполнена / не выполнена
```

| Статус | Описание |
|---|---|
| `черновик` | Ученик ещё заполняет |
| `на оценке` | Отправлена смотрящему, ждёт оценки сложности |
| `в процессе` | Оценена, идут чекины |
| `на проверке` | Ученик заявил о выполнении, смотрящий проверяет |
| `на голосовании` | Смотрящий подтвердил, идёт голосование |
| `выполнена` | Народ оценил, начислен LIT |
| `не выполнена` | Народ оценил как провал, или дедлайн истёк |
| `отклонена` | Арбитры признали цель невалидной |

### 6.4 Подача доказательств

Когда ученик считает цель выполненной:
1. Нажимает "Заявить о выполнении" в Mini App
2. Загружает доказательства (тип указан при постановке цели): фото, видео, ссылка, документ
3. Добавляет описание того, что сделано
4. Заявка уходит смотрящему

**Смотрящий** получает уведомление в боте и открывает в Mini App:
- Видит доказательства и описание
- Кнопки: `✅ Подтвердить` / `📝 Запросить дополнение`
- При запросе дополнения — пишет комментарий, что именно нужно добавить
- При подтверждении — цель уходит на голосование

**Важно:** смотрящий несёт ответственность за подтверждение. Если он подтвердит фейк и арбитры это выявят — он теряет LIT и рейтинг вместе с учеником.

### 6.5 Народное голосование

**Количество голосующих** зависит от оценки сложности смотрящего:

| Сложность | Голосующих |
|---|---|
| 1–3 (малое) | 5 человек |
| 4–6 (среднее) | 10 человек |
| 7–9 (большое) | 20 человек |
| 10 (эпик) | 35 человек |

**Механика:**
- Голоса раздаются рандомно активным пользователям (у которых есть статус арбитра или выполнены ≥ 3 цели)
- Результаты скрыты до закрытия голосования
- Нельзя самому найти нужное голосование — только рандомная выдача

**Таймер:** голосование закрывается через 3 дня с момента начала, даже если не набрано нужное количество голосов. Нужен минимум 1 голос, иначе голосование перезапускается.

**Два ползунка при голосовании (0–10):**

```
Ползунок 1: Цель и её выполнение
  → Насколько амбициозна цель? Насколько качественно выполнена?
  → 0 = цель невалидна или доказательства явно не соответствуют
  → 10 = отличная цель, доказана блестяще

Ползунок 2: Работа смотрящего
  → Насколько хорошо смотрящий описал и помог с целью? Честно ли подтвердил?
  → 0 = явно плохая работа / подтвердил фейк
  → 10 = отличная поддержка и честная оценка
```

Оба ползунка независимы. Обязательно двигать оба перед отправкой.

**Подсчёт результатов:**
- Медиана по ползунку 1 → определяет LIT ученика
- Медиана по ползунку 2 → определяет LIT смотрящего
- Судьи, попавшие в ±1 от медианы по обоим ползункам → получают фикс бонус LIT

## 7. LIT — внутренняя валюта

### 7.1 Начисление LIT

| Событие | Размер |
|---|---|
| Цель выполнена | медиана₁ × коэф_сложности × 0.6 |
| Был смотрящим | медиана₂ × коэф_сложности × 0.3 |
| Честно проголосовал (±1 от медианы) | фикс: 2 LIT за голос |
| Ставка + цель выполнена | ставка возвращается + бонус из фонда |
| Смотрящий получает долю штрафа провалившегося ученика | 30% от суммы штрафа |
| Был учителем — хорошие отзывы ученика | по тарифу уровня |

**Коэффициенты сложности (для расчёта LIT):**

| Сложность | Коэффициент |
|---|---|
| 1–3 | 1× |
| 4–6 | 3× |
| 7–9 | 8× |
| 10 | 20× |

Базовый LIT за оценку 10 при сложности 1 = ~6 LIT. При сложности 10 и оценке 10 = ~120 LIT.

### 7.2 Трата LIT

| Действие | Стоимость |
|---|---|
| Ставка на цель | любая сумма, добровольно |
| Буст заявки | 10 LIT → цель видят больше голосующих |
| Доступ к учителю | по тарифу уровня учителя |
| Создание группы | 50 LIT |
| Разблокировка бейджей | 5–20 LIT |

### 7.3 Механика ставки

Ученик при постановке цели может добавить ставку LIT (добровольно):

```
Цель выполнена и подтверждена → ставка возвращается + бонус из общего фонда
Цель провалена → ставка частично в фонд системы, часть смотрящему
```

Фонд системы = накапливается из:
- Проигранных ставок
- Платы за создание групп
- Бустов
- Платы за доступ к учителям (до передачи учителю)

Из фонда выдаются:
- Бонусы выполнившим с положительной ставкой
- Фикс бонусы честным судьям

### 7.4 Динамический курс LIT (будущее)

Курс LIT к реальным деньгам зависит от активности системы:
- Много выполненных целей → курс растёт
- Много провалов и неактива → курс падает
- Это создаёт коллективную мотивацию в успехе друг друга

В первой версии курс не реализуется — только виртуальная валюта.

## 8. Учителя

### 8.1 Стать учителем

Требования:
- Уровень ≥ 4
- Рейтинг ≥ 3.5
- Выполнено ≥ 5 целей в теме, которую хочет преподавать

Процесс:
1. Пользователь подаёт заявку в Mini App
2. Заполняет профиль учителя: тема, описание, что умеет, чем может помочь
3. **Испытательный срок:** первые 3 ученика бесплатно
4. После 3 учеников — рейтинг учителя виден всем, тарификация включается

### 8.2 Профиль учителя

В профиле учителя хранится:
- Тема обучения
- Описание подхода
- **История достижений:** все выполненные цели в этой теме (из истории как ученика)
- **Статистика как смотрящего:** сколько учеников довёл до результата
- Количество учеников
- Средний рейтинг от учеников
- Достижения (бейджи)
- Статус: ✅ Активен (≥ 4.0) / 🟡 На испытании / 🔴 Заморожен (<3.0)

### 8.3 Тарификация учителя (по уровням)

| Уровень пользователя | Стоимость занятия |
|---|---|
| 1–3 | бесплатно (испытательный срок) |
| 4 | 10 LIT |
| 5 | 15 LIT |
| 6 | 25 LIT |
| 7 | 35 LIT |
| 8 | 50 LIT |
| 9 | 70 LIT |
| 10 | 100–150 LIT |

Рейтинг внутри уровня двигает цену: рейтинг 5.0 = верхняя граница диапазона, рейтинг 3.5 = нижняя.

**Формула:** `цена = базовая_цена_уровня × (rating / 5.0)`

### 8.4 Поток оплаты учителя

```
Ученик выбирает учителя → оплачивает LIT из баланса
LIT блокируется в банке системы
         ↓
По итогу занятий и отзыва ученика:
         ↓
  Отзыв хороший (≥ 4.0)    Отзыв плохой (< 2.5)
  ↓                         ↓
Учитель получает            30% LIT возвращается
100% LIT из банка           ученику, учитель
                            получает 70%
```

### 8.5 Доска объявлений учителей

Отдельный раздел в Mini App:
- Список учителей с фильтрами (тема, уровень, рейтинг, цена)
- Карточка учителя: профиль, достижения, отзывы, цена
- Кнопка "Записаться" → списание LIT, создание связи ученик-учитель

## 9. Группы

### 9.1 Создание группы

Стоимость: 50 LIT (уходит в банк системы).

Создатель заполняет:
- Название и описание группы
- Тема / направление
- Максимальное количество участников (2–20)
- Открытая или по приглашению

### 9.2 Механика группы

- У каждого участника по-прежнему есть своя пара (смотрящий ↔ ученик)
- Группа — дополнительный слой: общий чат, общие цели, групповой прогресс
- Участники видят прогресс друг друга
- Достижение общей цели группы → групповой бонус LIT всем участникам

## 10. Система жалоб и арбитраж

### 10.1 Жалобы на цели

Любой пользователь видит публичные цели. Если цель кажется невалидной (неконкретной, нереальной, абсурдной):

1. Нажимает "Пожаловаться" → указывает причину
2. При 3+ жалобах → цель уходит на рассмотрение арбитрам
3. **Арбитр решает:**
   - Жалоба обоснована → цель возвращается автору на переформулирование, рейтинг автора −0.1, рейтинг смотрящего −0.05
   - Жалоба необоснована → жалобщик получает небольшой штраф к рейтингу
4. Каждый жалобщик, чья жалоба подтверждена арбитром → **+2 LIT** фикс бонус

### 10.2 Статус арбитра

Открывается автоматически при: уровень ≥ 6, рейтинг ≥ 4.0, выполнено ≥ 10 целей.

Арбитр:
- Получает рандомные жалобы на рассмотрение
- За каждое решение → +1 LIT
- Если его решение оспорено и отменено → −0.1 к рейтингу арбитра

Статус арбитра теряется при:
- Рейтинг < 3.5
- 3+ отменённых решения подряд

## 11. Интерфейс (Telegram Mini App)

### 11.1 Требования к адаптивности

**Обязательно работает на:**
- Мобильный (основной режим, 375px–430px)
- Планшет (768px+)
- ПК в Telegram Desktop (900px+)

Используй CSS Grid / Flexbox. Не использовать фиксированные пиксельные ширины для основных элементов.

### 11.2 Навигация Mini App

Нижняя панель навигации (мобайл) / боковая панель (ПК):

```
🎯 Мои цели      → список целей, добавить цель
👁 Слежу          → список моих учеников, их чекины
🏆 Рейтинг        → таблица лидеров, мой профиль
💰 LIT            → баланс, история транзакций
📚 Учителя        → доска объявлений учителей
```

### 11.3 Экраны и компоненты

#### Профиль пользователя
- Аватар из Telegram
- Имя, уровень (бейдж), рейтинг (звёзды 0–5)
- Баланс LIT
- Статистика: выполнено / провалено целей
- История достижений
- Кнопка "Стать учителем" (если доступно)

#### Постановка цели
- Форма с 6 полями (описана в разделе 6.1)
- Прогресс-бар заполнения
- Превью как будет выглядеть цель публично

#### Просмотр цели
- Статус (визуально выделен)
- Прогресс: сколько чекинов сделано
- Дедлайн: обратный отсчёт
- Кнопка "Заявить о выполнении"
- История чекинов

#### Страница голосования
- Доказательства (медиа, ссылки)
- Описание выполнения от ученика
- Ползунок 1: "Оцени цель и результат" (0–10) с якорными примерами
- Ползунок 2: "Оцени работу смотрящего" (0–10) с якорными примерами
- Кнопка "Отправить оценку" (активна только если двинуты оба ползунка)

#### Доска учителей
- Карточки с фильтрами
- Карточка: фото, имя, тема, рейтинг, цена, количество учеников
- Модальное окно с полным профилем

### 11.4 Дизайн-система

- Цвета: использовать CSS переменные Telegram (`--tg-theme-bg-color`, `--tg-theme-text-color` и т.д.) для автоматической поддержки светлой/тёмной темы
- Шрифт: системный (SF Pro / Roboto в зависимости от ОС)
- Кнопки: скруглённые (border-radius: 12px), достаточно крупные для тапа (min-height: 48px)
- Ползунки: стандартные HTML range input или кастомные (не меньше 44px touch area)

## 12. Telegram Bot

### 12.1 Команды бота

```
/start         → регистрация / главное меню
/goals         → мои цели
/checkin       → отправить чекин
/partner       → информация о партнёре
/balance       → баланс LIT
/help          → помощь
```

### 12.2 Уведомления

Бот отправляет уведомления при:

| Событие | Кому |
|---|---|
| Новый чекин от ученика | Смотрящему |
| Смотрящий подтвердил / запросил дополнение | Ученику |
| Партнёр не выходил 3 дня | Активному партнёру |
| Цель вышла на голосование | Ученику |
| Голосование завершено, LIT начислен | Ученику и смотрящему |
| Новая жалоба на рассмотрение | Арбитру |
| Жалоба на цель подтверждена | Автору цели |
| Новый ученик записался к учителю | Учителю |
| Матч найден (новый партнёр) | Обоим |
| Дедлайн цели завтра | Ученику |

### 12.3 Открытие Mini App из бота

Главная кнопка бота открывает Mini App. Дополнительно — inline кнопки в уведомлениях:
- "Открыть чекин" → Mini App на странице чекина
- "Посмотреть доказательства" → Mini App на странице голосования

## 13. База данных (PostgreSQL) — схема

```sql
-- Пользователи
users (
  id              BIGINT PRIMARY KEY,
  telegram_id     BIGINT UNIQUE NOT NULL,
  username        VARCHAR(255),
  display_name    VARCHAR(255),
  level           INT DEFAULT 1,
  rating          DECIMAL(3,2) DEFAULT 5.00,
  lit_balance     INT DEFAULT 0,
  is_teacher      BOOLEAN DEFAULT FALSE,
  teacher_rating  DECIMAL(3,2),
  is_arbitrator   BOOLEAN DEFAULT FALSE,
  freeze_count_30d INT DEFAULT 0,
  created_at      TIMESTAMP DEFAULT NOW()
)

-- Пары
pairs (
  id              BIGSERIAL PRIMARY KEY,
  watcher_id      BIGINT REFERENCES users(id),
  student_id      BIGINT REFERENCES users(id),
  status          VARCHAR(20) DEFAULT 'active', -- active | frozen | ended
  freeze_initiated_by BIGINT REFERENCES users(id),
  created_at      TIMESTAMP DEFAULT NOW(),
  ended_at        TIMESTAMP
)

-- Цели
goals (
  id              BIGSERIAL PRIMARY KEY,
  user_id         BIGINT REFERENCES users(id),
  pair_id         BIGINT REFERENCES pairs(id),
  title           VARCHAR(200) NOT NULL,
  success_criteria TEXT NOT NULL,
  motivation      TEXT,
  action_plan     TEXT,
  proof_type      VARCHAR(50), -- video | screenshot | link | document | combined
  deadline        DATE NOT NULL,
  difficulty      INT,          -- 0-10, выставляет смотрящий
  stake_lit       INT DEFAULT 0,
  status          VARCHAR(30) DEFAULT 'draft',
  created_at      TIMESTAMP DEFAULT NOW(),
  completed_at    TIMESTAMP
)

-- Чекины
checkins (
  id              BIGSERIAL PRIMARY KEY,
  goal_id         BIGINT REFERENCES goals(id),
  user_id         BIGINT REFERENCES users(id),
  content         TEXT,
  media_urls      TEXT[],
  confirmed       BOOLEAN DEFAULT FALSE,
  confirmed_at    TIMESTAMP,
  created_at      TIMESTAMP DEFAULT NOW()
)

-- Доказательства выполнения
proofs (
  id              BIGSERIAL PRIMARY KEY,
  goal_id         BIGINT REFERENCES goals(id),
  description     TEXT NOT NULL,
  media_urls      TEXT[],
  watcher_confirmed BOOLEAN,
  watcher_note    TEXT,
  submitted_at    TIMESTAMP DEFAULT NOW(),
  confirmed_at    TIMESTAMP
)

-- Голосования
votes (
  id              BIGSERIAL PRIMARY KEY,
  goal_id         BIGINT REFERENCES goals(id),
  voter_id        BIGINT REFERENCES users(id),
  score_goal      INT NOT NULL, -- 0-10
  score_watcher   INT NOT NULL, -- 0-10
  created_at      TIMESTAMP DEFAULT NOW()
)

-- Голосования (мета)
voting_sessions (
  id              BIGSERIAL PRIMARY KEY,
  goal_id         BIGINT REFERENCES goals(id) UNIQUE,
  required_votes  INT NOT NULL,
  votes_count     INT DEFAULT 0,
  deadline        TIMESTAMP NOT NULL,
  status          VARCHAR(20) DEFAULT 'open', -- open | closed
  median_goal     DECIMAL(3,1),
  median_watcher  DECIMAL(3,1),
  closed_at       TIMESTAMP
)

-- LIT транзакции
lit_transactions (
  id              BIGSERIAL PRIMARY KEY,
  user_id         BIGINT REFERENCES users(id),
  amount          INT NOT NULL, -- положительный или отрицательный
  type            VARCHAR(50), -- goal_reward | watch_reward | vote_bonus | stake_win | stake_loss | teacher_payment | etc
  related_id      BIGINT,      -- goal_id или другое
  note            TEXT,
  created_at      TIMESTAMP DEFAULT NOW()
)

-- Жалобы
reports (
  id              BIGSERIAL PRIMARY KEY,
  reporter_id     BIGINT REFERENCES users(id),
  goal_id         BIGINT REFERENCES goals(id),
  reason          TEXT NOT NULL,
  status          VARCHAR(20) DEFAULT 'pending', -- pending | approved | rejected
  arbitrator_id   BIGINT REFERENCES users(id),
  decision_note   TEXT,
  created_at      TIMESTAMP DEFAULT NOW(),
  resolved_at     TIMESTAMP
)

-- Учителя
teacher_profiles (
  id              BIGSERIAL PRIMARY KEY,
  user_id         BIGINT REFERENCES users(id) UNIQUE,
  topic           VARCHAR(200),
  description     TEXT,
  status          VARCHAR(20) DEFAULT 'trial', -- trial | active | frozen
  students_count  INT DEFAULT 0,
  trial_students  INT DEFAULT 0,
  created_at      TIMESTAMP DEFAULT NOW()
)

-- Ученик-учитель связи
teacher_students (
  id              BIGSERIAL PRIMARY KEY,
  teacher_id      BIGINT REFERENCES users(id),
  student_id      BIGINT REFERENCES users(id),
  status          VARCHAR(20) DEFAULT 'active',
  lit_amount      INT,
  student_rating  DECIMAL(3,2),
  created_at      TIMESTAMP DEFAULT NOW(),
  ended_at        TIMESTAMP
)

-- Группы
groups (
  id              BIGSERIAL PRIMARY KEY,
  creator_id      BIGINT REFERENCES users(id),
  name            VARCHAR(200),
  description     TEXT,
  topic           VARCHAR(200),
  is_public       BOOLEAN DEFAULT TRUE,
  max_members     INT DEFAULT 10,
  members_count   INT DEFAULT 1,
  created_at      TIMESTAMP DEFAULT NOW()
)

-- Участники групп
group_members (
  id              BIGSERIAL PRIMARY KEY,
  group_id        BIGINT REFERENCES groups(id),
  user_id         BIGINT REFERENCES users(id),
  role            VARCHAR(20) DEFAULT 'member', -- creator | member
  joined_at       TIMESTAMP DEFAULT NOW()
)
```

## 14. API — основные эндпоинты

```
POST   /api/auth/telegram         → авторизация через Telegram WebApp initData
GET    /api/users/me               → профиль текущего пользователя
GET    /api/users/:id              → публичный профиль

GET    /api/pairs/current          → текущая пара
POST   /api/pairs/change           → запросить смену пары
POST   /api/pairs/wait             → ждать замороженного партнёра
POST   /api/pairs/replace          → заменить замороженного партнёра

GET    /api/goals                  → мои цели
POST   /api/goals                  → создать цель
GET    /api/goals/:id              → цель по id
PUT    /api/goals/:id/difficulty   → смотрящий ставит сложность
POST   /api/goals/:id/proof        → подать доказательства
POST   /api/goals/:id/confirm      → смотрящий подтверждает
POST   /api/goals/:id/checkin      → сделать чекин

GET    /api/voting/next            → получить случайное голосование
POST   /api/voting/:id/vote        → проголосовать

GET    /api/teachers               → список учителей (с фильтрами)
GET    /api/teachers/:id           → профиль учителя
POST   /api/teachers/apply         → подать заявку на учителя
POST   /api/teachers/:id/hire      → записаться к учителю
POST   /api/teacher-students/:id/rate → оставить отзыв учителю

GET    /api/lit/balance            → текущий баланс
GET    /api/lit/history            → история транзакций

POST   /api/reports                → подать жалобу
GET    /api/reports/queue          → очередь жалоб для арбитра
POST   /api/reports/:id/decide     → решение арбитра

GET    /api/groups                 → список групп
POST   /api/groups                 → создать группу
POST   /api/groups/:id/join        → вступить в группу

GET    /api/leaderboard            → рейтинг пользователей
```

### Авторизация

Все запросы к API требуют заголовок `Authorization: Bearer <token>`.

Токен выдаётся при POST `/api/auth/telegram` — передаётся `initData` из Telegram Web App. Сервер верифицирует подпись через HMAC-SHA-256 с токеном бота.

## 16. Безопасность

- Все запросы к Telegram WebApp верифицируются через `initData` и HMAC
- Rate limiting: не более 60 запросов в минуту с одного Telegram ID
- Файлы доказательств: проверка MIME-типа, максимальный размер 50MB
- Валидация всех входных данных (zod / pydantic)
- SQL инъекции исключены через ORM + параметризованные запросы
- XSS: sanitize при сохранении текстового контента
